# log-canvas Requirements

## 概要
DLTログをホワイトボード型のキャンバスと組み合わせて、調査しながらメモ・マーキングができるログビューアー。  
スクリーンショット＋外部メモツールの行き来をなくし、一つのアプリで「読む・考える・記録する」を完結させる。

## ターゲットユーザー
- 一人で車載ログ（DLT形式）を解析するエンジニア
- 調査中に根拠付きのメモを残したいユーザー

## コアコンセプト
**「調査セッション」** として、ログ読解の過程（フィルタ状態・メモ・マーキング）をまるごと保存・再現できる。


## 機能要求

### MVP（最初に作る）

**ログビュー（左ペイン）**
- DLTファイルの読み込み・表示
- 基本的なフィルタリング（ECU、Service、Severity等）
- カラムの表示/非表示切り替え
- 行選択 → キャンバスへの送り出し（右クリック or ドラッグ）

**キャンバス（右ペイン）**
- 無限スクロール可能なキャンバス
- ログ行をカードとして貼り付け
- カードにメモを書ける（Markdown対応）
- カード間を矢印で繋げる
- カードに色付け

### MVP後に検討

- メモ付き行だけをログビューでハイライト表示
- 「メモのある行だけ表示」フィルタ
- フィールドレベルのハイライト（この値が重要、というマーキング）
- 調査セッションの保存・ロード
- チーム共有（将来）

## 技術スタック
- **フレームワーク**：Tauri
- **フロントエンド**：React + TypeScript (Vite) + Tailwind CSS
- **キャンバスライブラリ**：React Flow
- **バックエンド**：Rust
