# log-canvas ロードマップ

## 概要

`requirements.md` に定義された機能要求を、依存関係と開発効率を考慮して5フェーズに分割する。
現在はTauri + React + Rustのスキャフォールド状態であり、全フィーチャーは未実装。

---

## フェーズ一覧

| フェーズ | テーマ | 対象要求ID |
|---|---|---|
| Phase 1 | 基盤構築 | C-01〜05 |
| Phase 2 | DLTログビュー（基本表示） | LV-01〜04 |
| Phase 3 | ログビュー操作 | LV-05〜08 |
| Phase 4 | コメント（付箋） | CM-01〜06 |
| Phase 5 | プロジェクト保存・復元 | PM-01〜02 |

---

## Phase 1 — 基盤構築

**目的**: 全フィーチャーのベースとなる無限キャンバス環境を整える。

### 対象要求

| ID | 内容 |
|----|------|
| C-01 | 無限スクロール可能な2Dホワイトボードを表示できる |
| C-02 | マウスホイールまたはピンチ操作でズームイン/アウトできる |
| C-03 | ドラッグ操作でキャンバスをパン（視点移動）できる |
| C-04 | キャンバス上のオブジェクトを自由に移動できる |
| C-05 | 複数のログビューとコメントを同一キャンバス上に共存させられる |

### タスク

- React Flow + Tailwind CSS をインストール・設定
- 無限キャンバスコンポーネントの実装（パン・ズーム・背景グリッド）
- ノードのドラッグ移動とマルチノード共存の動作確認

---

## Phase 2 — DLTログビュー（基本表示）

**目的**: DLTファイルを読み込み、キャンバス上にリサイズ可能なカードとして表示する。

### 対象要求

| ID | 内容 |
|----|------|
| LV-01 | DLTファイルを読み込み、キャンバス上にカードとして表示できる |
| LV-02 | タイムスタンプ・ECU ID・App ID・Context ID・メッセージ等のカラムを持つ |
| LV-03 | ログビュー内を縦方向にスクロールできる |
| LV-04 | ログビューのサイズをリサイズハンドルで自由に変更できる |

### タスク

- サンプルDLTファイルジェネレーター（モノレポ内サブプロジェクト `tools/dlt-sample-gen/` として作成、Rustバイナリ）
- Rust: DLTバイナリファイルのパーサー実装
- Rust: `open_dlt_file` / `get_log_rows` コマンドを tauri-specta で公開
- フロントエンド: ログビューカードコンポーネント（仮想スクロール対応）
- フロントエンド: リサイズハンドルの実装

---

## Phase 3 — ログビュー操作

**目的**: ログ解析に必要な絞り込み・視覚化機能を追加する。

### 対象要求

| ID | 内容 |
|----|------|
| LV-05 | カラム単位でフィルタリング（値の一致・正規表現等）を適用できる |
| LV-06 | 任意のログ行をマーキング（色付きハイライト）できる |
| LV-07 | カラムの表示/非表示・順序を変更できる |
| LV-08 | その他一般的なDLT Viewerの機能を保有する |

### タスク

- フロントエンド: カラム単位のフィルタUI（値一致・正規表現）
- Rust: フィルタリング処理の実装（パフォーマンスを考慮してバックエンド側で処理）
- フロントエンド: 行ハイライト（色付きマーキング・複数色対応）
- フロントエンド: カラム表示/非表示・ドラッグ並べ替え

---

## Phase 4 — コメント（付箋）

**目的**: キャンバスへの自由なメモ機能と、ログ行への自動アンカー機能を実装する。

### 対象要求

| ID | 内容 |
|----|------|
| CM-01 | キャンバス上の任意の位置に付箋コメントを作成できる |
| CM-02 | 付箋コメントのテキストを編集・削除できる |
| CM-03 | ログ行の右端からXpx以内にコメントを配置すると、そのログ行に自動的に紐付けられる |
| CM-04 | 紐付けられたコメントはログ行のスクロールに追従して位置が固定される |
| CM-05 | 紐付けコメントの高さに合わせて対応するログ行の高さが自動で変化する |
| CM-06 | 紐付け済みコメントを行から切り離し、フリーなコメントに戻すことができる |

### タスク

- フロントエンド: フリー付箋コンポーネント（作成・テキスト編集・削除）
- フロントエンド: ログビュー右端からのXpx近接検出 → 自動アンカーロジック
- フロントエンド: アンカー済みコメントのスクロール追従（ログ行オフセットと同期）
- フロントエンド: アンカー行の高さ自動調整（コメント高さに追従）
- フロントエンド: アンカー解除UI（フリーコメントに戻す）

---

## Phase 5 — プロジェクト保存・復元

**目的**: 作業状態をファイルに保存し、後から完全に再開できるようにする。

### 対象要求

| ID | 内容 |
|----|------|
| PM-01 | キャンバス状態（ログビューの位置・サイズ・フィルタ、コメントの内容・位置・紐付け情報）をファイルに保存できる |
| PM-02 | 保存したプロジェクトファイルを読み込み、作業状態を復元できる |

### タスク

- キャンバス状態のシリアライズ形式を設計（JSONベースのプロジェクトファイル）
- Rust: ファイル書き込み・読み込みコマンドの実装
- フロントエンド: 保存・読み込みUI（メニューバーまたはショートカット）
- フィルタ設定・コメントアンカー情報を含む完全な状態の保存・復元検証

---

## 依存関係

```
Phase 1（キャンバス）
    └─ Phase 2（ログ表示）
            └─ Phase 3（フィルタ・マーキング）
            └─ Phase 4（コメント・アンカー）
                    └─ Phase 5（プロジェクト保存）
```

Phase 3 と Phase 4 は Phase 2 完了後に並行して着手可能。
